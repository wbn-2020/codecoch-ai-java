## 3. 题目筛选修复说明

管理端 `/admin/questions` 分页查询现在支持：

- `status`：按题目启用/禁用状态筛选。
- `tagId`：先通过 `question_tag_relation` 查询匹配的 `question_id`，再对 `question` 分页查询追加 `id in (...)` 条件。

如果某个 `tagId` 没有关联题目，后端会追加一个不可能命中的题目 ID 条件，确保返回空分页结果，而不是误返回全部题目。

用户端 `/questions` 仍然强制叠加 `status=1`，不会因为管理端新增 `status` 筛选而放开禁用题。用户端仍可使用 `tagId`、`categoryId`、`difficulty`、`keyword` 等条件筛选启用题。

## 4. Prompt 筛选处理说明

Prompt 模板分页接口已有分页基础能力，补充筛选属于小范围改动，因此本轮已在后端补齐：

- `keyword`：匹配模板名称或模板内容。
- `scene`：按 Prompt 场景精确筛选。
- `status`：按启用/停用状态筛选。

接口仍为 `/admin/ai/prompts`，只是在原 `pageNo/pageSize` 基础上增加可选查询参数，不改变返回结构。
