## 验证与验收边界（重要）
- 本机环境不满足启动本地服务的条件，无法在本地起服务做端到端测试。不要尝试起服务，也不要声称已在本地完成运行时/端到端验收。
- 你需要本地跑通的是【不依赖运行服务】的自动化测试：
  - 后端：mvn -pl codecoachai-ai -am test -B 通过
  - 前端：npm run type-check && npm run test:unit:run && npm run check:quality 通过
- 后端 CI 不覆盖 dev-260703 分支，所以以上本地自测必须自己跑，别指望 CI。
- 补齐文档 §6 要求的单测：AI 成功 / AI 抛异常降级 / 低样本 LOW / listReviews 刷新回读
- 前端新增测试放 tests/unit/ 下（src/**/*.test.ts 当前不被 vitest 收集）
- 真正的端到端验收（起服务、连真实 AI 链路、页面点验复盘五分区和降级标签）由人工发布到测试环境后进行，不在你本次交付范围内。请在交付说明里明确「哪些已通过本地单测、哪些留待测试环境验收」。
