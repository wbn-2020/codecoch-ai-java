## 0. 一句话目标

把 `AgentGrowthServiceImpl.generateReview` 的复盘内容从"规则拼接 + 调试字符串"升级为"调用 `AiService` 生成结构化每日复盘（事实/限制/偏移/调整/下一步 + 自然语言总结）"，并放开 `v4Growth` feature flag，让用户在复盘中心看到真正的 AI 复盘；AI 不可用时优雅降级回现有规则逻辑并明确标注"降级复盘"。

### 3.3 `AiServiceImpl.generateAgentReview` 实现

严格照 §1.3 的 `analyzeSkillGap` 结构：

1. `validateGenerateAgentReviewDTO`：`dto != null`、`userId != null`。其余允许为空（低样本也要能生成）。
2. `promptRenderService.render(SCENE_AGENT_REVIEW_GENERATE, agentReviewPromptContent(), variables(dto))`。
   - 新增 scene 常量 `SCENE_AGENT_REVIEW_GENERATE`（参照现有 `SCENE_SKILL_GAP_ANALYZE` 的定义位置与命名规范）。
3. mock 分支：`aiProperties.getMockEnabled()` 为真时返回 `mockAgentReviewJson()`（构造一份结构完整的中文样例）。
4. 真实分支：`callAndLog(...)` → `parseAgentReviewJson(rawResponse)`。
5. 组装 VO，`setAiCallLogId(logId)`、`setRawResponse(firstText(rawResponse, resultJson))`。
6. catch：`saveLog(...失败...)` 后 `throw toBusinessException(ex)`。

**Prompt 要求（写进 prompt 内容）**：

- 系统角色：资深求职教练，基于当日 Agent 任务事实做每日复盘。
- 强约束返回 JSON，字段与 `GenerateAgentReviewVO` 对齐（summary/facts/limits/drifts/adjustments/nextActions）。
- 必须遵守项目"可信边界"约定：
  - facts 只陈述已记录的任务事实，不得编造外部求职结果（offer/面试通过等）。
  - 任务样本 `taskCount < 3` 或 `doneCount < 2` 时，limits 必须显式说明"样本不足，结论仅为弱调整信号"。
  - 不得给出与数据矛盾的乐观结论。
- 全中文输出。

**JSON 解析 `parseAgentReviewJson`**：参照现有 `parseSkillGapAnalyzeJson`，容错处理 markdown code fence、字段缺失（缺失字段回退空列表）。

### 3.4 `AgentGrowthServiceImpl.generateReview` 改造

注入 `private final AiService aiService;`。

改动点（第 106-119 行附近）：

```java
// 旧：
// List<String> nextActions = nextActions(tasks, done, skipped, todo);
// review.setSummary(summary(tasks.size(), done, skipped, todo, readinessScore));
// review.setNextActionsJson(writeJson(nextActions));

// 新：
AgentReviewNarrative narrative = buildNarrative(userId, targetJobId, date, tasks,
        done, skipped, todo, completionRate, agentSuccessRate, readinessScore);
review.setSummary(narrative.summary());
review.setNextActionsJson(writeJson(narrative.nextActions()));
review.setAiCallLogId(narrative.aiCallLogId());
review.setReviewJson(writeJson(reviewPayload(date, tasks, completionRate, agentSuccessRate, narrative)));
```

新增私有方法 `buildNarrative(...)`：

```java
private AgentReviewNarrative buildNarrative(...) {
    boolean trusted = tasks.size() >= MIN_TRUSTED_TASK_COUNT && done >= MIN_TRUSTED_DONE_TASK_COUNT;
    String confidence = trusted ? "HIGH" : "LOW";
    try {
        GenerateAgentReviewDTO dto = toAiDto(...); // 组装精简任务/技能
        GenerateAgentReviewVO ai = aiService.generateAgentReview(dto);
        return new AgentReviewNarrative(
            firstText(ai.getSummary(), ruleSummary(...)),
            nonEmpty(ai.getFacts(), ruleFacts(...)),
            nonEmpty(ai.getLimits(), ruleLimits(trusted)),
            nonEmpty(ai.getDriftReasons(), ruleDrifts(done, skipped, todo)),
            nonEmpty(ai.getAdjustments(), ruleAdjustments(...)),
            nonEmpty(ai.getNextActions(), nextActions(tasks, done, skipped, todo)),
            false, confidence, ai.getAiCallLogId());
    } catch (RuntimeException ex) {
        // AI 不可用：降级为现有规则逻辑，标 fallback=true
        log.warn("agent review AI generation failed, fallback to rule-based. userId={}, date={}", userId, date, ex);
        return new AgentReviewNarrative(
            ruleSummary(...), ruleFacts(...), ruleLimits(trusted),
            ruleDrifts(done, skipped, todo), ruleAdjustments(...),
            nextActions(tasks, done, skipped, todo),
            true, confidence, null);
    }
}
```

- `AgentReviewNarrative`：一个 record/内部类，承载 summary/facts/limits/drifts/adjustments/nextActions/fallback/confidenceLevel/aiCallLogId。
- `ruleSummary/ruleFacts/ruleLimits/ruleDrifts/ruleAdjustments`：把**当前前端本地兜底逻辑等价搬到后端**作为规则降级实现（见 `agentLoopAdapter.ts` 第 37-55 行），保证降级时后端产出和以前前端兜底一致。
- 保留原 `summary(...)` 调试串方法可删除或改造为 `ruleSummary` 的中文版；**不要再返回 `"Agent review: total=..."` 这种调试串**。
- 现有的 `saveReadiness / saveSkillSnapshots / createReviewMemory` 调用顺序与逻辑**保持不变**。

### 3.5 持久化与回读

存储分两类，务必区分：

- **标量字段 `confidence_level`、`fallback` → 存真实列**（见 §3.6 迁移）。
  - `AgentReview` entity 新增 `private String confidenceLevel;`、`private Boolean fallback;`（MyBatis-Plus 默认驼峰↔下划线映射到 `confidence_level`/`fallback`，无需额外注解）。
  - `generateReview` 落库时 `review.setConfidenceLevel(narrative.confidenceLevel())`、`review.setFallback(narrative.fallback())`。
- **数组正文 `facts/limits/driftReasons/adjustments` + `summary` → 继续存 `reviewJson`**。
  - `reviewPayload(...)` 扩展：在原有 `date/taskCount/doneCount/completionRate/agentSuccessRate/topSkills` 基础上，新增 `narrative` 子对象，存 `summary/facts/limits/driftReasons/adjustments`。
- `toReviewVO(...)`（第 479-496 行）扩展：
  - `facts/limits/driftReasons/adjustments`：解析 `review.getReviewJson()` 中的 `narrative` 回填；解析失败留空（前端本地兜底）。
  - `confidenceLevel/fallback`：直接从 entity 列读取回填。
- 这样 `generateReview` 返回值和 `listReviews` 刷新读取**走同一套 `toReviewVO`**，保证一致。

### 3.6 数据库迁移

文件：`codecoach-ai-java/sql/migration/V4_072__agent_review_ai_narrative_fields.sql`

> 命名沿用现有序号（最新为 `V4_071`）。**必须幂等**——参照 `V4_070__career_import_row_mediumtext.sql` 用 `information_schema` 判断列是否已存在再决定是否 `ALTER`，避免重复执行报错。

要点：

- 给 `agent_review` 加两列：
  - `confidence_level VARCHAR(16) NULL`（值 `HIGH`/`LOW`）
  - `fallback TINYINT NOT NULL DEFAULT 0`（0=正常/AI，1=降级）
- **历史行回填**：`fallback` 有 `DEFAULT 0` 自动兜底；`confidence_level` 允许 NULL，历史行留空即可（`toReviewVO` 对 null 不回填，前端本地兜底）。无需额外 UPDATE。
- 幂等写法（每列一个 `information_schema.columns` 判断 + `PREPARE/EXECUTE`），照抄 `V4_070` 结构。
- **同步基线**：`sql/init.sql` 里 `agent_review` 建表语句（若存在完整建表基线）也要加这两列，保证全新初始化和迁移路径一致。Codex 需确认 `init.sql` 是否含 `agent_review`，含则同步。

## 5. 降级与可信边界（务必遵守项目约定）

| 场景 | 行为 |
| --- | --- |
| `mockEnabled=true` | 走 mock JSON，正常展示，`fallback=false`（属正常 mock 链路，非降级）。可选：mock 时 confidenceLevel 照常按样本量算 |
| AI 调用异常/超时 | catch 后降级规则逻辑，`fallback=true`，前端显示"降级复盘"黄标 |
| 样本不足（task<3 或 done<2） | `confidenceLevel=LOW`，limits 明确"弱调整信号"，前端显示 LOW 标签 |
| AI 返回字段缺失 | 逐字段回退规则兜底，不整体失败 |

- 复盘正文**不得**声称任何未被任务数据支撑的外部结果。
- 降级复盘也要能正常落库、能被 `listReviews` 读到。
