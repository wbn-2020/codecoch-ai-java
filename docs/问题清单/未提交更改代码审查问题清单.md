## P0：codecoachai-task 编译失败

- 文件：`codecoachai-task/src/main/java/com/codecoachai/task/controller/AdminTaskController.java`
- 位置：`stats()` 方法
- 问题：`asyncTaskMapper.selectMaps(...)` 返回 `List<Map<String, Object>>`，当前代码赋值给 `List<Object>`，导致编译失败。
- 影响：`codecoachai-task` 模块无法通过编译，相关任务管理、通知、MQ 消费等变更不能交付。
- 建议：
  - 将返回类型调整为明确的统计 VO/DTO，或改为 `List<Map<String, Object>>`。
  - 查询中补充实际计数字段，例如 `status, count(*) as count`，否则当前 group by 只返回状态值，不返回数量。
- 验证：
  - `mvn -pl codecoachai-task -am compile -DskipTests`

## P1：通知接口存在跨用户数据访问风险

- 文件：`codecoachai-task/src/main/java/com/codecoachai/task/controller/NotificationController.java`
- 位置：
  - `myNotifications(...)`
  - `unreadCount(...)`
  - `markRead(...)`
  - `markAllRead(...)`
- 问题：
  - 列表、未读数、全部已读接口信任请求参数 `userId`。
  - 单条已读接口只按通知 `id` 更新，没有附加 `user_id` 归属条件。
- 影响：普通用户可能通过篡改 `userId` 或通知 `id` 读取、统计、修改其他用户通知，违反用户数据隔离要求。
- 建议：
  - 从 `LoginUserContext` 或网关透传的登录用户上下文获取当前用户 ID。
  - 移除用户侧接口中的 `userId` 请求参数。
  - 所有查询和更新都附加 `Notification::getUserId == currentUserId` 条件。
  - 如果保留管理员查询能力，应放到 `/admin/**` 并走管理员权限。
- 验证：
  - 用户 A 不能读取用户 B 的通知列表和未读数。
  - 用户 A 不能通过用户 B 的通知 ID 标记已读。

## P1：搜索同步消费失败后重试会被幂等标记跳过

- 文件：`codecoachai-search/src/main/java/com/codecoachai/search/consumer/SearchSyncConsumer.java`
- 位置：`onMessage(...)`
- 问题：Redis 幂等 key 在 Elasticsearch 写入或删除之前就已经写入。若 ES 操作抛异常，RocketMQ 重试时会因为 Redis key 已存在直接返回。
- 影响：搜索索引更新或删除可能永久丢失，业务数据与 ES 索引不一致。
- 建议：
  - 将幂等成功标记移动到 ES 操作成功之后。
  - 或在 catch 中删除本次写入的幂等 key，让 RocketMQ 重试可以重新执行。
  - 如果需要并发去重，可采用 processing/success 双状态或短 TTL 锁，避免失败被误判为已消费。
- 验证：
  - 模拟 ES 写入失败后重试，确认第二次会重新执行 ES 操作。
  - 模拟重复成功消息，确认不会重复写入异常数据。

## P1：批量 AI 出题任务未持久化题目却标记成功

- 文件：`codecoachai-task/src/main/java/com/codecoachai/task/consumer/QuestionGenerateConsumer.java`
- 位置：AI 返回后到 `asyncTaskService.markSuccess(...)`
- 问题：消费者调用 AI 得到题目草稿后，只统计 `draftCount` 并标记异步任务成功，没有调用 question-service inner 接口写入 `question_draft` 或对应审核池。
- 影响：后台看到任务成功，但没有可审核、可使用的题目草稿，形成假成功。
- 建议：
  - 增加 question-service inner 写入接口或复用已有批量草稿保存能力。
  - 在写入成功后再调用 `markSuccess`。
  - 写入失败应作为可重试异常抛出，不能吞掉或标记成功。
- 验证：
  - 触发批量出题任务后，AI 返回的草稿能在题目审核池或草稿表查询到。
  - 模拟 question-service 写入失败，任务状态应失败或进入重试，而不是成功。
